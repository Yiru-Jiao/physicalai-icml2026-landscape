3. PhysicalAI Topic Map

本文使用的工作定义是：

Physical AI = Perception + World/Dynamics Modeling + Action/Control + Embodiment + Evaluation/Safety

这个定义刻意把 PhysicalAI 从单纯 “robotics” 扩展到更宽的 physical-world intelligence：只要一个系统需要理解、预测或作用于真实/模拟物理世界，它就落入本报告关注范围。

Perception / 3D / Spatial

把 image/video 转成可用于 action 的 geometry、scene、object、motion representation。

World / Dynamics Model

学习 action-conditioned transition、long-horizon consistency 或 rollout imagination。

Policy / Action Decoding

把 instruction、visual context、latent state 转成 continuous 或 discrete action chunks。

Simulation / Evaluation / Safety

提供 data generation、closed-loop benchmark、safe failure mode 与 deployment-oriented evaluation。

5. Technical Clusters

5.1 World Models & Planning

World model 在 ICML 2026 的 PhysicalAI 相关工作中不再只是 video generator，而是被用作 policy training、synthetic rollout、planning 与 policy evaluation 的基础接口。VLAW、DreamDojo、GRASP、Jumpy World Models、WorldCompass 等工作共同指向一个趋势：模型需要学习的不只是视觉未来，而是可被 action 改变、可被 planning 使用、可被 evaluation 检验的 dynamics。

判断：World model 的核心难点不是生成质量，而是 physical fidelity、action consistency、long-horizon error accumulation 与 policy-induced distribution shift。

5.2 VLA & Embodied Policy

VLA 相关工作开始从 “language-conditioned policy” 向更低延迟、更强 spatial grounding、更贴近 continuous control 的形式移动。Any3D-VLA 强调 3D point-cloud grounding；LaRA-VLA 与 LaST_0 强调 latent reasoning；Discrete Diffusion VLA 强调 action chunk generation；NeurVLA 强调 failure correction/prevention。

判断：VLA 的核心矛盾是 language token reasoning 与 continuous physical control 的表示错位。未来更可能是 hybrid：high-level language reasoning + low-level latent/action-space dynamics。

5.3 Robotics Manipulation & Control

DexMachina、DLO-Lab、RoboTwin 2.0、DICE-RL、CaP-X 等工作体现出 ICML 对 manipulation/control 的关注不只停留在 policy architecture，而是进入 contact、bimanual manipulation、deformable object、simulation scalability、policy refinement 等更真实的物理难点。

判断：Manipulation 是 PhysicalAI 最直接的试验场，因为它强迫模型面对 contact discontinuity、partial observability、geometry precision 与 closed-loop correction。

5.4 Simulation / Physics / Data Generation

SceneSmith、SafeLab、GeoPT、RoboTwin 2.0、DLO-Lab 等工作说明 simulation 不再只是低成本替代品，而是训练、评估、安全检验和 data engine 的组成部分。

判断：PhysicalAI 的 scaling law 很可能不是单纯参数量 scaling，而是 real data × simulation data × world-model rollout × evaluator feedback 的组合 scaling。

5.5 3D / Spatial / Video Dynamics

3D-RFT、Any3D-VLA、Flex-Forcing、VideoFlexTok、LUVE 等工作代表了 PhysicalAI 的感知基础层。它们不一定直接输出 action，但对 scene geometry、motion continuity、long-video consistency、3D reasoning 的改进，会影响下游 embodied policy。

判断：Video/3D 工作只有在能转化为 state abstraction、dynamics prior 或 action-relevant representation 时，才真正成为 PhysicalAI 的核心组成。

7. Research Judgement

Trend 1 — From static perception to action-conditioned modeling PhysicalAI 的关键不只是 “看懂”，而是 “知道动作会如何改变世界”。这使 world model、action-conditioned video、planning 与 closed-loop evaluation 变得更重要。
Trend 2 — VLA is moving below language tokens Language 仍是 instruction interface，但连续控制更需要 latent action、diffusion action、3D grounding 与 fast feedback。
Trend 3 — Simulation and evaluation become algorithmic infrastructure 真实物理数据昂贵且稀疏，因此 simulation-ready scene、domain randomization、generative benchmark、safe lab 不只是工具，而是训练闭环的一部分。
Trend 4 — Real-world deployment gap remains large 许多工作在 LIBERO、RoboTwin、simulation 或 constrained real-robot setting 上验证。真实长期、开放环境、safety-critical deployment 仍是未解决问题。
Trend 5 — Autonomous driving signal is indirect but relevant ICML 2026 中直接以 autonomous driving 为核心的 PhysicalAI 信号弱于 robotics，但 world model、video dynamics、planning、closed-loop evaluation 对 driving foundation policy 仍有直接启发。
综合判断：ICML 2026 的 PhysicalAI 还没有形成像 LLM scaling 那样清晰的单一路线，但已经形成一个较稳定的技术堆栈：spatial representation → world/dynamics model → action/policy → simulation/evaluation → safety/deployment。未来真正有竞争力的系统大概率不是单点模型，而是能把这些层闭环起来的 data-model-evaluator pipeline。

8. What to Watch Next

1. World model 的 fidelity

需要区分 perceptual realism 与 physical correctness。对 robot/driving 来说，后者更关键。

2. Latent reasoning 与 control frequency

VLA 要解决高层 reasoning 慢、低层 control 快的时间尺度冲突。

3. Action representation

continuous trajectory、discrete action tokens、diffusion chunks、latent actions 各有 bias。这个选择会影响 RL/post-training 与 deployment。

4. Evaluation beyond success rate

PhysicalAI 需要 failure taxonomy、risk-sensitive metrics、closed-loop robustness 和 recoverability。

5. Sim2Real data engine

未来核心不是 “sim 或 real”，而是如何把 sim、real、world-model rollout 与 evaluator 组合成高质量 data flywheel。

6. Safety / privacy / governance

Embodied AI 会采集和使用真实空间、人体、动作与场景数据，privacy-utility trade-off 会从边缘议题变成系统约束。
